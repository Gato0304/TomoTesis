\textbf{Rocio A. Rojas S.}

\begin{large}
\textbf{Diseño de un modelo de emulación para un sistema de segundo orden
	basado en el método $"$Hardware in the Loop$"$.}
\end{large}
\end{center}

\noindent%
\textbf{Tutor Académico: Mercedes Arocha. Tesis.
Caracas, Universidad Central de Venezuela. Facultad de Ingeniería.
Escuela de Ingeniería Eléctrica. Mención Electrónica, Computación y Control. Año 2024.}

\noindent
\textbf{Palabras clave:} Hardware in the Loop, HIL, emulación, control, controlador, sistema de segundo orden. \\[1ex]

\noindent \textbf{Resumen.} 

%El principal objetivo de este trabajo fue el diseño de un modelo de emulación de un sistema de segundo orden aplicando el método "Hardware in the loop". Esto es, en un sistema de control real, sustituir una planta física por una computadora y usar un controlador físico. Por ello, como resultado se diseñó un programa de emulación de una planta de segundo orden basado en Python el cual incluye una interfaz gráfica. Dicho programa implementa un modelo discretizado de un sistema de segundo orden, que responde a estímulos que vienen dados por un controlador que ejecuta un algoritmo de control por ubicación de polos usando realimentación de estados y un observador de orden reducido implementado con una placa de desarrollo Espressif ESP32 en lenguaje C, con el objetivo de validar el desempeño de un sistema de control, sin necesidad de realizar pruebas físicas que puedan resultar costosas, peligrosas o difíciles de alcanzar.


El objetivo principal de este trabajo consistió en diseñar un modelo de emulación de un sistema de segundo orden utilizando el método "Hardware in the loop". Este enfoque aplicado en sistemas de control moderno, consiste en simular un sistema físico real mediante el uso de hardware y software. Esto requiere conectar una parte del sistema de control real, llamada "hardware", a un modelo matemático del sistema bajo prueba, llamado "simulador". Como resultado, se diseñó un programa basado en Python que emula una planta de segundo orden y cuenta con una interfaz gráfica. Este programa implementa un modelo discretizado del sistema, el cual responde a estímulos generados por un controlador que utiliza un algoritmo de control por ubicación de polos con realimentación de estados y un observador de orden reducido implementado con una placa de desarrollo Espressif ESP32 en lenguaje C. El propósito de esta emulación es proporcionar un entorno de prueba seguro y eficiente para evaluar el rendimiento y las características de un controlador antes de implementarlo en el sistema real, ya que al utilizar el hardware real en lugar de solo una simulación computacional, se pueden simular con mayor precisión las condiciones reales del sistema, lo que permite un análisis más realista de la respuesta del controlador y la interacción con otros componentes.
