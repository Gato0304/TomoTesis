En la actualidad, existe la necesidad en la industria de crear técnicas que permitan realizar pruebas de validación a los diferentes dispositivos que constituyen un sistema de control. Por este motivo, surge un método llamado $"$Hardware in the Loop$"$ que consiste en una técnica donde a través de un controlador conectado a un sistema de pruebas, con el fin de evaluar el desempeño del controlador. Disminuyendo de esta manera el costo y el tiempo que demanda realizar los distintos escenarios de pruebas en el sistema de interés, ya que el controlador está comprendido como un hardware físico y los ensayos se realizan sobre una emulación.

Los sistemas que utilizan el método $"$Hardware in the Loop$"$ normalmente están integrados por un entorno de emulación y una unidad electrónica de control que representan al dispositivo que se encuentra bajo prueba. De este modo, para realizar la emulación de la planta se puede utilizar una computadora o un sistema embebido como un PLC, FPGA o DSP, entre otros. Este entorno de emulación será conectado a otro sistema físico que representará al controlador,
el cual opcionalmente puede tener un conjunto de actuadores y sensores que servirían para optimizar el sistema de control. 

Es importante destacar que para poder obtener resultados que aporten valor, es necesario una alta calidad en el software de simulación. Este software debe combinarse con hardware que pueda permitir la inserción de fallas y la habilidad de probar escenarios reales.

En este trabajo se utilizó el método $"$Hardware in the Loop$"$ con el objetivo de validar el desempeño de un sistema de segundo orden, con el interés de reforzar los conocimientos de las asignaturas de la Unidad Docente Control de la Escuela de Ingeniería Eléctrica de la Universidad Central de Venezuela.
