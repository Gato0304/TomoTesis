\section{Controlador digital}

Se tiene el controlador digital que consta de un proceso de multitareas dividas entre dos núcleos del procesador; dos de estas tareas se dedican al cálculo de los valores y se ejecutan en simultáneo a la tarea de monitoreo de la comunicación. 

La ecuación que da forma al algoritmo es:

%El algoritmo implementa la ecuación \ref{ec:dinamicareducida2}

\begin{equation*}
	u(k+1)= -[(k_{1}+GK_{2})y(k+1) + G_{y}y(k)+ G_{u}u(k)]
	%\label{ec:dinamicareducida2}
\end{equation*}
De acuerdo a lo indicado en el diagrama de la figura 
 \begin{figure}[H]
 	\centering
 	\includegraphics[width=1.0\linewidth ]{img/diagramasimplificado}
 	\caption{Diagrama de bloques del sistema simplificado}
 	%\label{fig:diagramasimplificado}
 \end{figure}
 
\begin{itemize}
	\item aproxTask: Tarea usada para aproximar los cálculos, alojada en el núcleo 0.
	\item controllerTask: Tarea usada para dar la respuesta del controlador, alojada en el núcleo 0.
	\item rx\_task: Tarea siempre activa, usada para el monitoreo de la comunicación serial, alojada en el núcleo 1.
\end{itemize}

La tarea de aproximación obedece a la acción del bloque sumador en la realimentación del lazo de control. Esto sugiere que para estimar la salida del controlador en tiempo real, se tiene que estimar aproximadamente la salida del sistema mediante la solución de una ecuación no lineal.

Se usó el método de la secante de análisis numérico, el cual consiste en aproximar la raíz de una ecuación de forma iterativa a partir de dos valores iniciales.

Para estimar la raíz de $f(x) = 0$, se define la relación de recurrencia \cite{secantefuente}:

\begin{equation}\label{metodosecante}
	x_{k} = x_{k-1} - f(x_{k-1}) \dfrac{x_{k-1} - x_{k-2}}{f(x_{k-1}) - f(x_{k-2})}
\end{equation}


Teniendo en cuenta que la función de transferencia del sistema en lazo cerrado es:

\begin{equation*}
	\frac{Y(z)}{R(z)} =\frac{U_{e}G_{discreta}}{1 + U_{e}U_{y}G_{discreta}}
\end{equation*}

Se puede reinterpretar de esta manera:

\begin{equation}\label{metodosecante1}
	E(z) = R(z) - U_{y}Y(z)
\end{equation}

\begin{equation}\label{metodosecante2}
	Y(z) = U_{e}G_{discreta}(z)E(z)
\end{equation}

Sustituyendo \ref{metodosecante1} en \ref{metodosecante2}: 

\begin{equation}\label{metodosecante3}
	Y(z) = U_{e}G_{discreta}(z)(R(z) - U_{y}Y(z))
\end{equation}



Entonces, para aplicar el método se debe estimar $Y(z)$ tal que  

\begin{equation}\label{metodosecante4}
	f(Y(z)) =  Y(z) - U_{e}G_{discreta}(z)(R(z) - U_{y}Y(z)) = 0
\end{equation}

para dar solución a la ecuación \ref{metodosecante4}, se tiene el siguiente fragmento de código el cuál interpreta la ecuación \ref{metodosecante}:

\begin{lstlisting}[caption = {Solución de la ecuación \ref{metodosecante4} por el método de aproximación de la secante},captionpos=b,label={codedisrangobs},language=C]
	
	double secanteAprox()
	{
		tControlParams.sec_x0 = 0;
		tControlParams.sec_x1 = 1;
		tControlParams.sec_x2 = 0;
		for (int i = 0; i < 20; i++)
		{
			tControlParams.sec_x2 =
			tControlParams.sec_x0 - (plantaIterativa(tControl.ref, tControlParams.sec_x0) * (tControlParams.sec_x1 - tControlParams.sec_x0) /(plantaIterativa(tControl.ref, tControlParams.sec_x1) - plantaIterativa(tControl.ref, tControlParams.sec_x0)));
			if (fabs(tControlParams.sec_x2 - tControlParams.sec_x1) < 0.0001)
			break;
			tControlParams.sec_x0 = tControlParams.sec_x1;
			tControlParams.sec_x1 = tControlParams.sec_x2;
		}
		
		return tControlParams.sec_x2;
	}
\end{lstlisting}

En él se observa que la iteración tiene un límite de 20 ciclos con una convergencia del 0.01\%

Para dar la respuesta del controlador basada en esta estimación:
\begin{lstlisting}[caption = {Cálculo de la respuesta en tiempo real del controlador digital basado en la respuesta de la planta},captionpos=b,label={codedisrangobs},language=C]
	
	void controlador(double plantaDouble)
	{
		char *ptr;
		char sendBuff[100];
		#ifdef DEBUG
		printf("plantaDouble: %3.9f\n\n sec_x2: %3.9f\n\n", plantaDouble, tControlParams.sec_x2);
		#endif
		double now = Huy(plantaDouble);
		double fError = tControl.ref - now;
		double val = Hue(fError);
		long entero = val * FLOAT_SCALER;
		tControlParams.planta_x1 = (Gp.b1 * val - Gp.a1 * plantaDouble) + tControlParams.planta_x2;
		tControlParams.planta_x2 = (Gp.b0 * val - Gp.a0 * plantaDouble);
		sprintf(sendBuff, "%ld\n", entero);
		sendData(sendBuff);
		#ifdef DEBUG
		printf("sendData: %ld\n\n", entero);
		#endif
	}
\end{lstlisting}

Esta función se ejecuta una vez se recibe un nuevo valor de respuesta desde la interfaz de emulación y la estimación por el método secante para dar respuesta del controlador a través del puerto serial.
